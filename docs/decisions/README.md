# Architecture Decision Records

Significant architectural decisions for Skills.

| ADR | Status | Summary |
|-----|--------|---------|
| [001-skill-consolidation](001-skill-consolidation.md) | Accepted | Consolidate 48 granular skills to 7 composite skills |
| [002-consequences-based-learning](002-consequences-based-learning.md) | Accepted | Learn from outcomes, not static instructions |
| [003-three-layer-architecture](003-three-layer-architecture.md) | Accepted | Progressive enhancement: CLI → hooks → gateway |

## ADR Format

Each ADR follows the standard format:
- **Status**: Proposed, Accepted, Deprecated, Superseded
- **Context**: Why this decision was needed
- **Decision**: What was decided
- **Consequences**: Positive, negative, and neutral impacts
